## Descripción

El autocompletar sugiere al usuario resultados coincidentes mientras escribe. Por cada carácter introducido, se envía una consulta al servidor para obtener los resultados apropiados. Esto puede resultar costoso.

Es preferible utilizar la entrada asistida, que consiste en guiar al usuario a través de un conjunto de información y sugerencias gestionadas localmente, lo que reduce el número de interacciones con el servidor.

Minimizar el impacto del autocompletado con optimizaciones sencillas: un retardo de unas décimas de segundo entre la adición de un carácter y la consulta, lanzar la consulta sólo después de que se haya introducido un determinado número de caracteres, limitar el número de resultados mostrados, etc.

Para minimizar las idas y vueltas, si los datos presentados al usuario son suficientemente limitados, puede incluirlos directamente en su código HTML y utilizar el elemento nativo \<datalist>.

## Ejemplo

El elemento \<datalist> una ida y vuelta con el servidor.

```html
<label for="ice-cream-choice">Choose a flavor:[1]</label>
<input list="ice-cream-flavors" id="ice-cream-choice" name="ice-cream-choice" />

<datalist id="ice-cream-flavors">
	<option value="Chocolate">
	<option value="Coconut">
	<option value="Mint">
	<option value="Strawberry">
	<option value="Vanilla">
</datalist>
```
